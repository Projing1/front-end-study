# HTML5简介

  • 广义的HTML5是指HTML、CSS和JavaScript在内的一套技术组合
  • 主要作用：减少网页浏览器对于需要插件的丰富性网络应用服务，如Adobe Flash、Microsoft Silverlight 与Oracle JavaFX的需求；
    HTML5设计目的 是在移动设备上支持多媒体，为支持这一点：
  • 引入新的多媒体元素：如：audio、video；
  • 引入图形绘制：canvas；
  • 引入新的语义结构元素：footer 、article、aside、nav等；
  • 引进新的表单控件：calendar、date、time、email、url、search等；

# HTML5新增特性

新元素、新属性、支持css3、2D/3D制图、本地存储、Web应用、video支持、audio支持；

# HTML5对我们的作用：

  • 互联网领域：为下一代web提供全新的框架和平台，使web能够轻松实现类似桌面应用的应用体验；
  • 编程人员：丰富的标签体系，可取代那些完成较简单任务的插件，降低应用开发的技术门槛；
  • SEO：有利于搜索引擎抓取和索引网站内容，提供更多的功能、更好的体验；
  • 企业：改变企业网络广告的模式与分布，助力传统企业IT应用移动化，帮助企业构建应用平台；

# 浏览器支持：

Safari、Chrome、opera、Firefox都支持，IE9开始支持。

# HTML5的新特性

# HTML5基础

新增标签：header（网站头部）、nav（导航栏）、section（类似div）、aside（文件侧栏）、article（文章、文本内容）、footer（网站脚本）……；
新增表单输入框类型：type="email"(限制用户输入必须是Email类型)、type="url"(限制用户输入必须是URL类型)、type="date"(自动生成一个日期控件)、type="number"(限制用户输入必须是数字类型)、type="range"(产生一个进度条的表单)、type="search"(产生一个搜索意义的表单)、type="color"(生成一个颜色选择表单)
新增输入框的属性： placehoder:占位文本，体验更佳；autofocus ：自动获取焦点；autocomplete ：提交一次后下次自动补全；multipe ：配合file控件实现多选；
HTML5制图 & Canvas 新增的元素<canbas>只是图形的容器，必须通过脚本来绘制图形。你可以通过多种方法使用<canvas>绘制路径——盒、圆、字符以及添加图像；
HTML5多媒体 & video ：提供播放、暂停和音量控件来控制视频；提供width和height属性控制视频尺寸，标签中间可以用<source>来添加多个视频源。<video>标签间插入的内容是不支持video元素的浏览器显示的</video>
HTML5多媒体 & audio ：音频播放；标签中间可以用<source>来添加多个音频源 HTML5的web存储功能：将数据存储到用户电脑上，缓解服务器压力，提高体验。具体分为：

  • 会话存储（sessionStorage）生命周期为关闭浏览器窗口，同一个窗口下数据可共享。
  • 本地存储（localStorage）永久生效，除非手动删除，可多窗口共享。

# 拖放释放功能

定义：拖放是HTML5 常见特性，即抓取对象以后拖到另一个位置，并且任何元素都能拖放。在元素上设置draggable="true"即可拖动。
在拖动目标上会触发的事件（源对象）：

  • ondragstart （源对象开始拖动元素时触发）
  • ondrag 源对象正在拖动时触发
  • ondragend 源对象完成拖动后触发
    在目标对象上会触发的事件：
  • ondragenter 目标对象进入其容器范围内时触发
  • ondragover 目标对象进入源对象范围内拖动时触发；
  • ondragleave 源对象离开器容器范围内时触发；
  • ondrap 释放鼠标键时触发

# HTML5选择器、拖放、History

# HTML5选择器

HTML选择器是基于选择器规则进行查找，并且可以更方便的获取DOM元素，同时编写原生JavaScript更加方便。
新增的选择：

  • querySelector：根据选择器规则，返回相匹配的第一个元素，没有找到返回null；
  • querySlectorAll ：根据选择规则，返回文档中所有符合要求的元素，并且返回NodeList对象。 CSS基础选择器：
  • 元素选择器：标记名{/* 声明块 */}（所有与该标记名匹配的元素，都将应用声明块中的规则。）
  • 类选择器：**.*类名{/ 声明块 */}（所有class属性为指定类名的元素，都将应用声明块中的规则，其中类型名可以写多个，方便代码复用所有标签都可有class属性）
  • id选择器：#id值{/* 声明块 */} <h1 id="属性值">(属性值为指定的元素，都将应用声明块中的规则，在同一个HTML文档中，元素的id值必须唯一)
  • 通配符选择器： *+{声明块}
  • 并集选择器/组合选择器 : 元素或类或ID+ ","+ 元素或类或id+ "," +元素或类或id "," + {声明块} eg：
/* 元素选择器 */
div{} 

/* 类选择器 */
.my-class{}

/* id选择器 */
#id{}

/* 通配符选择器 */
*{}

/* 并集选择器 */
div , .my-class , #id {}

/* 组合选择器 */
div.my-class{}
div .my-class{}

css的层次选择器

/* 子集选择器 ：父级元素名称+ “>” +子级元素名称+ {声明块} */
div>p{color:red}

/* 兄弟选择器：A元素+ "+" + B元素名称+ {声明块}*/
.main_span + p{}

/* 后代选择器：祖先元素名称+空格+后代元素名称+ {声明块} */
div p{}

/* 通用选择器：同级元素A+"~"+同级元素B + {声明块} */
.main_span ~ p{}

css的动态伪类选择器; hover/link/active/visited四类 link：未访问过的链接。
hover：鼠标悬停的。
active：鼠标点击时。
visited：点击后。
focus：输入框等被选中时。

css的结构伪类选择器；

/* 选中第一个 */
p:first-child{}

/* 选中最后一个 */
p:last-child{}

/* 选中奇数项 ：元素名称+":nth-child(odd)+{}"*/
p:nth-child(odd){}

/* 选中偶数项：元素名称+":nth-child(even)+{}" */
p:nth-child(even){}

TIP

nth-child()：可以传参，如odd，even等，也可以是单个整数，表达式等， n从0开始取整数，直到最大值，例如：选中前三个：-n+3 ；

# History

# 常用方法：

  • back：前往上一页，用户点击浏览器左上角的返回按钮模拟此方法；

WARNING

注：当浏览器会话历史记录处于第一页时调用此方法没有效果，而且也不会报错；等价于history.go(-1)

  • forword : 在浏览器历史记录里前往下一页，用户可点击浏览器左上角的前进按钮模拟此方法，等价于history.go(1)

WARNING

当浏览器历史栈处于最顶端时（当前页面处于最后一页）调用此方法没有效果也不报错。

  • go :通过当前页面的相对位置从浏览器历史记录（会话记录）加载页面。
    history是历史状态管理，允许操作浏览器曾经在标签页或者框架里访问的会话历史记录。
    新增特性：
名称 描述
pushState 每执行一次都会增加一条历史记录，浏览器在返回时，就不会返回前一个页面，并且不会刷新浏览器。
replaceState 用来修改当前的历史记录，而不是创建一个新的历史记录，点击返回按钮照样返回上一个页面。
onpopstate 它是popstate在Window对象上的事件，pushState或者replaceState不会触发popstate事件，只会在点击后退、前进按钮或者调用history.back() ,history.forword(),history.go()

pushState语法：

history.pushState(state,title,url);

参数说明：

  • state：状态对象，记录状态通过pushState创建新的历史记录并且存储起来（Firefox存储有大小限制，必选小于640K，否则会抛出异常）
  • title ：标题（目前被忽略）：Firefox目前忽略这个参数，但未来可能用到，传递一个空字符串在这里是安全的，而在将来这是不安全的，二选一的话，你可以为跳转的state传递一个短标题。
  • 需要修改的地址 重新定义新的url记录，浏览器不会立即加载，可以是相对路径并且必须要和当前url同源，否则会抛出异常；执行完浏览器会更新，但是不会真的向服务器发送请求。

# 跨文档消息通讯

# 概念

跨文档指的是跨HTML文件，不是document对象。
跨文档通讯的定义：跨文档消息（cross-document messaging），有时候也简称XDM，指的是来自不同域的页面间传递消息。使用场景：内嵌框架（iframe）和其父文档相互进行数据传输或者通讯。
HTML5跨文档通讯：
核心：postMessage()方法；

window.postMessage(message,origin,[transfer])
  • message:消息对象：1.最早作为“永远都是字符串”来实现的，后来参数定义修改为：允许传入任何数据结构。2.它是将要发送到其他window的数据，将会被结构化克隆算法序列化，这意味着你可以不受限制的将数据对象安全的传送给目标窗口，而无需自己序列化。

WARNING

建议传递的参数类型为字符串，可以通过先转换为字符串再传递的方式。

  • origin：域信息；1、指定框架中的文档来源的域，如来源匹配，消息会传递到内嵌框架中，否则postMessage()什么也不做。这一限制可以避免窗口中的位置在你不知道的情况下发生改变。
  • transfer : 转移消息对象（可选参数）；是一串和message同时传递的Transferable对象，这些对象的所有权将转移给消息的接收方，而发送一方将不再保有所有权。
    message回调的event事件对象：
  • event.data : postMessage传输过来的第一个参数，通常是字符串类型数据；
  • event.origin : postMessage传输过来的第二个参数，是用来匹配消息来源方的域；
  • event.source : 来源方目标文档的window引用，通常用作单次握手操作回应的数据。
  • event.ports : 是MessageEvent接口的特有属性，用于获取消息端口集合对象，通常用于（ChannelMessage对象）；

# 地理信息、本地存储、离线存储

# 地理信息

定义：定位用户的位置，使用HTML5 的Geolocation API用于获的用户的地理位置（需要用户同意，否则用户位置信息不可用）；
判断是否支持地理信息： 通过navigator.geolocation对象检测，可以判断当前设备是否支持地理信息。
相关方法：

  • getCurrentPosition()方法来获得用户的位置;
navigator.geolocation.getCurrentPosition(success,error,options)

+++ success:成功得到位置信息时的回调函数，使用Position对象作为唯一参数； +++ error：[可选参数]，获取位置信息失败后的回调函数，使用PositionError对象作为唯一的参数； +++ options ： PositionOptions对象

# web存储

web存储API提供了localStorage（本地存储）和sessionStorage（会话存储）两个存储对象来对网页的数据进行增删改查操作；
localStorage用于长久保存整个网站的数据，保存的数据无过期时间，直到手动删除；
sessionStorage用于临时保存同一窗口（或标签页）的数据，在关闭窗口或标签页之后会删除这些数据。
localStorage常用方法：

//存入数据方法：
localStorage.setItem("key","value");

// 查询数据方法
localStorage.getItem("key");

// 删除数据方法
localStorage.removeItem("key");

// 清楚全部localStorage数据
locaStorage.clear();

# 离线存储

定义：HTML5引入了应用程序缓存，这意味着WEb应用可进行缓存，并可在没有因特网连接时进行访问。
离线存储的优势：

  • 离线浏览：即当用户网络断开时，可以继续访问页面；
  • 访问速度快： 将资源缓存到本地，已缓存资源加载更快；
  • 减少服务器负载： 浏览器只会从服务器下载更新过的或者更改过的资源；

cache Manifest基础：

  • 每个指定了mainfest的页面在用户对其访问时都会被缓存；
  • 如果未指定mainfest属性，则页面不会被缓存；
  • mainfest文件的建议文件扩展名是：".appcache";

mainfest文件实例

mainfest文件完整结构：

  • CACHE MAINFEST：此标题下列出的文件将在首次下载后进行缓存，CACHE表示缓存的文件，使用相对路径；
  • NETWORK：此标题下列出的文件需要与服务器连接，并且不会被缓存；
  • FALLBACK：标题下列出的文件规定当页面无法访问时的回退页面；
  • 使用#进行注释；
    eg：
CACHE MAINFEST
# 2019-09-01 V1.0.0
CACHE:
style/index.css
js/index.js

NETWORK:
login.php

FALLBACK:
/html5/404.html

TIP

如何使用mainfest ：在html标签上设置mainfest="<文件名>.appcache";
编写<文件名>.appcache文件；

# 音频与视频

# HTML5之前的多媒体解决方案：

HTML5之前不存在一项旨在网页上播放音频和视频的标准；
大多数音频和视频是通过插件（比如：Flash）来播放的，然而，并非所有浏览器都拥有同样的插件。
HTML5 规定了在网页嵌入音频与视频元素的标准；

# HTML5的解决方案：

HTML5规定了一种通过audio元素来包含音频的标准方法， 该元素能够播放声音文件或者音频流；
规定了一种通过video元素来包含视频的标准方法，该元素能够播放视频文件或者视频流；
Flv.js:
简介：HTML5Flash（FLV）播放器，纯原生JavaScript开发，由哔哩哔哩开源。
工作原理：默认，将FLV文件流转码复用为ISO BMFF（MP4碎片）片段，然后通过MEdia SourceExtensions将MP4片段汇进浏览器。
使用ECMAScript6编写，通过babel compiler编译成ECMAScript5 。使用Browserify打包。
主要作用：让HTML5能够支持FLV格式的视频。

浏览器全兼容视频格式解决方案：
video.js 优势:

  • 开源免费，很容易在GitHub上获取最新代码。
  • 简单易用，只需要几秒钟就能架起一个视频播放页面。
  • 几乎兼容所有浏览器，且优先使用HTML5，在不支持的浏览器中会自动使用flash播放，
  • 界面可定制，纯JavaScript和css打造，说明文档详细。

# canvas

使用场景：可以使用<canvas>提供的脚本API绘制丰富的图表，制作图片构图或者制作简单的动画。 兼容性：ie9以前的不支持。通过canvas.getContext方法来判断是否支持canvas；

# 矩形（Rectangular）

无论绘制空心实心矩形还是清除矩形区域，四个参数分别是x，y ，width ， height ；

var ctx = canvas.getContext("2d");
ctx.fillRect(25,25,,100,100); //实心矩形；
ctx.clearRect(45 ,45 ,60,60); //清除矩形区域；
ctx.strokeRect(50,50,50,50); //空心矩形

# 圆弧

x , y 为圆心点的以radius为半径的圆弧（圆），从startAngle开始到endAngle结束来生成；

// 参数为：x,y,radius,startAngle ,endAngle ,anticlockwise;
// 参数anticlockwise为一个布尔值，为true时，是逆时针方向，否则顺时针方向。（默认顺时针）
ctx.arc(75,75,50,0,2*Math.PI,true);
ctx.stroke();

# 路径（Paths）

图形的基本元素是路径，路径是通过不同颜色和宽度的线段或曲线相连形成的不同形状点的集合；

ctx.beginPath(); //开始路径 
ctx.moveTo(75,50); //画笔移动到75,50；
ctx.lineTo(100,75);  //从上个点往100,75绘制直线；
ctx.lineTO(45,125);  //再从上个点往45,125绘制直线；
ctx.stroke();

# canvas2d色彩（colors）

填充样式（FillStyle）：如果想要给图形上色，有两个重要的属性可以做到，fillStyle 和strokeStyle。

// 设置填充颜色，按照设置的颜色绘制一个空心矩形；
ctx.fillStyle = "#FD0";
ctx.fillRect(0,0,75,75);

轮廓样式（StrokeStyle） fillStyle用于填充颜色，strokeStyle用于描边颜色的设置，只要是符合W3c的颜色格式都支持；

ctx.strokeStyle = "rgb(0,255,255);"

透明度（Transparency） ：除了可以绘制实色图形，我们还可以用canvas的globalAlpha来绘制半透明图形；

# 贝塞尔曲线

二次和三次贝塞尔曲线都十分有用，一般用来绘制复杂有规律的图形；

  • quadraticCurveTo(cp1x,cp1y,x,y); 绘制二次贝塞尔曲线，cp1x,cp1y为一个控制点，x,y为一个结束点。
  • bezierCurveTo(cp1x,cp1y,cp2x,cp2y,x,y);绘制三次贝塞尔曲线，cp1x,cp1y为一个控制点，cp2x,cp2y为控制点2；x,y为结束点

用来缓存或记录绘画命令，以便快速地回顾路径； Path2D API有另一个强大的特点——使用SVG path data来初始化canvas上的路径。

# 状态、变形以及动画

canvas的状态控制：绘制复杂图形时必不可少的两个方法：

  • save() : 保存画布（canvas）的所有状态，Canvas的状态就是当前画面应用的所有样式和变形的一个快照。
  • restore() : save和restore方法是用来保存和回复canvas状态的，都没有参数。

TIP

canvas状态存储在栈中，每当save()方法被调用后，当前的状态就被推送到栈中保存。
一个绘画状态包括：
（1）、当前应用的变形（即移动，旋转和缩放，见下：） （2）、strokeStyle、fillStyle、globalAlpha、lineWidth、lineCap、lineJoin、miterLimit等等属性值；
（3）、当前的裁切路径（clipping path），
可以调用任意多次save方法；每次调用restore方法，上一个保存的状态就从栈中弹出，所有设定恢复。

变形的相关API：

  • 移动（Translating）：translate(x,y),接收两个参数，x是左右偏移量，y是上下偏移量；
  • 旋转（Rotating）：rotate(angle),这个方法只接收一个参数：旋转的角度（angle），他是顺时针的方向，以弧度为单位的值。
  • 缩放（Scaling）：scale(x,y),参数都是实数，可为负数，x为水平缩放，y为垂直缩放，如果比1小，会缩小图形，如果比1打，会放大图形，默认值为1；
  • 变形（Transforms）：transform(m11,m12,m21,m22,dx,dy):这个方法是将当前的变形矩阵乘上一个基于自身参数的矩阵；

动画的基本步骤：
可以通过以下的步骤来画出一帧：

  • 清空canvas： 除非接下来要画的内容会完全充满canvas（例如背景图），否则需要清空所有，最简单的方法就是用clearReact方法；
  • 保存canvas状态： 如果需要改变一些会改变canvas状态的设置（样式、变形之类的）,又要在每画一帧之时都是原始状态的话，需要先保存一下；
  • 绘制动画图形（animated shapes）：这一步才是重绘动画帧；
  • 恢复canvas状态：如果已经保存了canvas的状态，可以先恢复它，然后重绘下一帧。

# 弹性盒子模型

# 弹性盒子是什么以及作用：

1、弹性盒子是CSS3的一种新的布局模式；
2、CSS3弹性盒子（flexible Box或flXbox），是一种当页面需要适应不同的屏幕大小以及设备类型时，确保元素拥有恰当的行为的布局方式；
3、引入弹性盒布局模型的目的是提供一种更加有效的方式来对一个容器中的字元素进行排列，对齐和分配空白空间。

# 重要属性：

  • flex-direction：主轴方向（即元素排列的方向）。
  • justify-content：主轴元素的对齐方式。
  • align-items ： 定义元素在交叉轴上如何对齐。
  • flx-wrap ：规定元素的换行方式。
  • align-content：定义了多根轴线的对齐方式，如果项目只有一根轴线，该属性不起作用。

# touch事件

  • touchstart：当在屏幕上按下手指时触发
  • touchmove： 当在屏幕上移动手指时触发
  • touchend ： 当在屏幕上抬起手指时触发
  • touchcancel ： 当一些更高级别的事件发生的时候（如电话接入或者弹出信息）会取消当前的touch操作，即触发touchcancel。一般会在touchcancel时暂停游戏、存档等；
